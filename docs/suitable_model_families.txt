SUITABLE MODEL FAMILIES FOR THIS PROJECT
========================================

Reviewed against primary model documentation: 2026-05-26.

Selection rule
--------------
This repository benchmarks a model by supplying a still image plus a text
prompt and evaluating its generated text. This list therefore includes only
models that support image input and text output. Text-only LLMs, image
generators, embedding models, audio-only models, and video-only models are
excluded.

Status labels:
  IMPLEMENTED = a model wrapper is present under models/
  INCLUDED*   = a wrapper is present, but the model is optional in the
                default benchmark suite

Year meaning:
  Release year is the model-family/checkpoint release year stated by the
  publisher, or the year of an official dated snapshot/model-card/release
  announcement.

Notes:
  - All models listed here support image input and text output.
  - A compact, useful default API panel would be gpt-4o, gpt-4.1, gpt-5.2,
    gpt-5.4, gpt-5.4-mini, and gpt-5.5.
  - Include gpt-5 and gpt-5.1 only for historical trend analysis.
  - Avoid relying on gpt-5.3-chat-latest for reproducible long-term results:
    it is an alias for a ChatGPT model rather than the primary frontier line,
    and its official page does not publish a dated snapshot/release year.
  - The existing OpenAI wrapper already sends image and text through the
    Responses API, but GPT-5-family integration should not unconditionally
    send temperature; use options supported by the selected API model.

Notes:
  - LLaVA-NeXT (also called LLaVA 1.6) was released in January 2024 and is
    specifically relevant to this project because it improves OCR and visual
    reasoning on image tasks.
  - LLaVA-OneVision was released in 2024 and covers single-image as well as
    multi-image/video settings; its image checkpoints fit this benchmark.
  - The 7B OneVision checkpoint remains the lowest-friction missing addition,
    since a 72B OneVision wrapper is already implemented.
  - The 4B LLaVA-OneVision-1.5 checkpoint gives one newer representative
    without duplicating the practical size tier too heavily; it
    may require a different loading path (`trust_remote_code=True`) than the
    existing Hugging Face OneVision wrapper.
  - Specialized visual-prompt, multi-image interleave, single-image tuning,
    and very large NeXT variants are omitted to keep this a focused shortlist.
    Video checkpoints are also omitted because the current benchmark contract
    is for still images.

HIGH-PRIORITY MODELS FOR BENCHMARK RUNS
---------------------------------------
1. gpt-5.5 (2026)
   Current high-end API baseline.

2. gpt-5.4-mini (2026)
   Lower-cost API baseline suitable for running the full benchmark suite.

3. Qwen/Qwen2.5-VL-7B-Instruct (2025)
   Straightforward intermediate-size addition to an already supported family.

4. llava-hf/llava-onevision-qwen2-7b-ov-hf (2024)
   Practical local OneVision alternative to the represented 72B checkpoint.

5. lmms-lab/LLaVA-OneVision-1.5-4B-Instruct (2025)
   Newer compact LLaVA family baseline for single-image evaluation.

6. Qwen/Qwen3.5-4B (2026)
   New current-generation compact model with straightforward image prompting.

7. google/gemma-4-E4B-it (2026)
   Current compact-to-medium Google multimodal baseline.

8. openbmb/MiniCPM-V-4.6 (2026)
   Recent efficiency-focused image understanding model.

9. OpenGVLab/InternVL3-8B (2025)
   Newer comparison against the implemented InternVL2.5 model.
